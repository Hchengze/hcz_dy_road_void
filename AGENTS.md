# AGENTS.md

## 一、项目定位

本项目是一个面向“道路地下空洞探测”的本地科研原型项目，主要研究场景为：

* 道路一侧或道路附近布设 DAS 光纤；
* 另一侧或道路沿线使用锤击震源激发浅层瑞雷波；
* 地下可能存在空洞、脱空、松散区、管沟、管线、弱夹层、破碎带等异常体；
* 通过正演模拟、DAS 类响应、绕射/散射特征识别、多炮联合定位和后续反演，研究空洞位置、埋深和形态的识别能力。

本项目不是工业级商业软件，也不是完整复刻成熟地震波模拟平台。项目目标是逐步形成一个可运行、可解释、可验证、可学习的科研级算法平台。

所有新增功能都必须服务于以下主线：

```text
道路地下模型
  -> 多炮正演数据
  -> DAS 类接收响应
  -> 绕射/散射特征识别
  -> 多炮联合定位
  -> 局部全波场验证
  -> 逐步反演 refinement
```

不能为了增加功能数量而让主线变散。

## 二、长期技术目标

本项目最终希望具备以下科研能力：

### 1. 更真实的道路地下模型

逐步从简单均匀介质和单一空洞，发展到包含以下内容的道路地下场景：

* 路面层；
* 基层；
* 路基；
* 近地表低速层；
* 管沟；
* 管线；
* 空洞；
* 脱空；
* 松散区；
* 条带弱异常；
* 多异常体组合。

模型需要支持平面图、纵剖面、横剖面和简化三维展示。模型越复杂，越要说明其物理含义和近似边界。

### 2. 更真实的正演数据

正演能力应分层推进：

```text
第一层：快速运动学正演
第二层：多散射点 / 面采样 / 体采样近似
第三层：局部三维 elastic FDTD 验证
第四层：2D FEM / SEM / BEM 科研原型
第五层：逐步探索 FWI refinement
```

现有运动学多散射点模型不是错误，它是快速近似层。后续高级正演方法应补充它，而不是粗暴替代它。

### 3. 更接近 DAS 的接收模拟

当前 DAS 相关结果只能称为 DAS-like approximation。后续可逐步增强：

* 沿光纤方向差分；
* 应变近似；
* 应变率近似；
* gauge length 平滑；
* 通道耦合差异；
* 弱耦合段；
* 坏道；
* 交通相干噪声；
* 脉冲噪声；
* 带限震源；
* 多炮数据组织。

除非完整模拟了光纤方向、耦合、仪器响应、gauge length 和解调机制，否则不能声称是真实 DAS 仪器模拟。

### 4. 绕射/散射识别

不能只依靠肉眼看图。应逐步建立可解释的绕射/散射识别能力，包括：

* 直达波压制；
* 包络属性；
* 能量属性；
* 局部相干性；
* 多炮一致性；
* S-D-G 理论曲线叠加；
* 绕射候选评分；
* 低置信度提示。

识别结果必须进入后续定位和报告，不能只输出一张孤立属性图。

### 5. 多炮联合定位与误差评估

定位能力应从单炮或单异常体演示，逐步发展为多炮联合评估：

* 多炮联合扫描；
* x、y、depth 三维定位；
* y-depth 耦合分析；
* 误差统计；
* 置信度；
* 不确定性范围；
* 真值对比；
* 多异常体情况下的识别限制。

定位结果不能只给“最佳点”，还要给误差、置信度和失败原因。

### 6. 局部全波场验证

默认 workflow 可以继续使用快速运动学模型，但必须逐步使用局部全波场方法验证其合理性。

局部全波场验证包括：

* 小尺度 elastic3d；
* 体属性异常体；
* 局部裁剪模型；
* 震源、接收点、异常体是否落入局部模型范围的检查；
* 与运动学结果的对比。

elastic3d 是局部科研验证工具，不是默认全道路工业正演。

### 7. FEM / SEM / BEM / FWI 的推进方向

这些方法要逐步推进，不要跳步包装。

推荐路线：

```text
FEM：
1D 标量波教学原型
  -> 2D 标量波
  -> 2D SH 波
  -> 2D P-SV 弹性波
  -> 与 FDTD 对比
  -> 再考虑 3D 原型

SEM：
1D 谱元教学原型
  -> 2D 标量 SEM
  -> 2D 弹性 SEM
  -> 与 FDTD / FEM benchmark 对比
  -> 再考虑复杂模型

BEM：
边界积分思想演示
  -> 2D Helmholtz Green 函数
  -> 圆形空洞散射
  -> 边界条件对比
  -> 再考虑更复杂空洞边界

FWI：
misfit 曲线
  -> misfit map
  -> 有限差分梯度检查
  -> 单参数反演
  -> 双参数反演
  -> 小模型 refinement
  -> 后续再考虑伴随状态法
```

在正演求解器、数据残差、边界条件和梯度检查没有可信验证前，不得声称实现了完整 FWI。

## 三、默认 workflow 主链条

默认 workflow 是项目主线。所有重要算法结果都应尽量进入同一条链：

```text
统一配置
  -> 道路地下场景
  -> 几何与速度模型
  -> 多炮合成数据
  -> DAS 类响应
  -> 绕射/散射属性
  -> 多炮联合定位
  -> 波场示意
  -> 局部全波场验证接口
  -> 研究报告
```

新增算法时必须检查它是否应该同步影响：

* 地下模型图；
* 速度模型图；
* 正演炮集；
* DAS-like 炮集；
* 绕射属性；
* 定位结果；
* 波场示意；
* 误差图；
* 研究报告；
* 参数记录；
* 输出清单。

不能出现“某个算法推进了，但其它步骤仍然使用旧默认参数或旧数据”的情况。

---

## 四、参数一致性规则

项目中只能有一条主要参数链。

当用户修改道路宽度、道路长度、光纤间距、炮点间距、速度模式、层速度、异常体位置、异常体形状、扫描范围等参数时，相关流程必须同步更新。

必须避免：

* 几何图使用一套参数；
* 正演使用另一套参数；
* 扫描使用默认参数；
* 波场示意使用旧速度；
* 报告写入旧异常体；
* dataset 和 diffraction 使用不同数据。

如果某个参数不适用于某个模块，需要明确提示，而不是静默忽略。

例如：

* workflow 参数不一定能直接用于 elastic3d 全尺度模型；
* elastic3d 是局部小模型，需要检查异常体是否落入网格；
* layered-effective 速度不是完整分层弹性波传播；
* DAS-like 响应不是完整仪器模拟。

## 五、输出规则

默认 workflow 应输出完整有效结果，但不能重复、不能分裂、不能混乱。

默认 workflow 输出应集中在：

```text
outputs/workflow/
```

不应再默认生成：

```text
outputs/local_debug/
outputs/wavefield/
```

wavefield 是 workflow 的一个步骤，不是独立结果体系。

输出原则：

1. 默认输出应尽量完整；
2. 同一结果不要重复保存多份；
3. 文件命名要能体现流程顺序；
4. 每次运行应生成 output_manifest；
5. 每次运行应保存 run_parameters；
6. 重要结果应进入 research_report；
7. clean-output 只能清理当前 workflow 输出，不得误删 numerics、elastic3d_validation 等独立结果。

如果某个结果没有生成，应在报告中说明原因。

## 六、波场图件规则

当前 workflow 中的 wavefield 默认是运动学示意，不是完整弹性波场。

### x-y 平面波场

x-y 平面图是合理的，因为道路、光纤和炮点主要位于地表平面。它用于展示地表平面上的瑞雷波传播关系。

但必须明确标注：

```text
x-y 平面运动学波场示意
```

不能让用户误以为这是完整三维弹性波场。

### 3D kinematic wavefield

3D 波场示意只能表示三维几何、等时面、震源、异常体、接收线之间的空间关系。

必须明确标注：

```text
三维运动学波场示意，不是完整弹性波场
```

不得伪造真实分层反射、折射、模式转换或复杂弹性散射。

### 真正三维波场

真正三维弹性波场应归入：

```text
elastic3d
FDTD
FEM
SEM
```

并且必须经过稳定性、边界条件和图件检查。

## 七、速度模型规则

速度模式必须说清楚。

```text
uniform：
使用单一瑞雷波速度 VR。

layered-effective：
使用层状近地表速度计算等效瑞雷波速度 VR_eff。
```

layered-effective 不是完整 Rayleigh 频散反演，也不是完整弹性波传播。它只是当前运动学正演和扫描中的等效速度近似。

当速度参数改变时，必须同步影响：

* 速度模型图；
* 正演走时；
* 扫描速度；
* 波场示意标题；
* dataset metadata；
* research_report。

不能出现速度图显示分层，但正演或波场仍使用默认均匀速度的情况。

## 八、几何图件规则

坐标定义必须统一：

```text
x：沿道路 / 沿光纤方向
y：横穿道路方向
z：深度，向下为正
```

`01_geometry_plan_sections.png` 应包含：

```text
1. x-y 平面图；
2. x-z 沿道路剖面；
3. y-z 横穿道路剖面。
```

y-z 横剖面必须满足：

* 横轴为 y；
* 纵轴为 z 或 depth；
* z 向下为正；
* 光纤位于 y=0；
* 锤击线位于 y=road_width；
* 异常体位于 cavity_y 和 cavity_depth 对应位置。

不得把 y-z 和 z-y 混用。

## 十、注释规则

本项目也是学习项目。核心算法代码需要教学型中文注释。

注释重点：

* 参数单位；
* 物理含义；
* 数学公式和代码变量的对应关系；
* 近似假设；
* 适用范围；
* 常见错误；
* 图件含义；
* 与其它步骤的关系。

不需要给 import、空行、简单 return 写无意义注释。

注释的目标是帮助用户理解算法，而不是机械地把每行代码翻译成中文。

## 十一、测试和人工检查规则

每次重要修改后必须至少运行：

```bash
python -m compileall -q road_void examples main.py
python -m pytest -q
```

涉及 workflow 的修改，还应检查：

```bash
python main.py workflow --no-save
python main.py workflow --save --clean-output
```

涉及 elastic3d 的修改，还应检查：

```bash
python main.py elastic3d --no-save
python main.py elastic-validate --no-save
```

涉及 numerics 的修改，还应检查：

```bash
python main.py numerics-compare --no-save
```

仅 pytest 通过不代表完成。涉及图件和物理意义的修改，必须人工查看输出图，并说明检查结论。

## 十五、最重要的判断标准

本项目每一轮开发都要问：

```text
这个修改是否让道路空洞正演、识别、定位、反演主线更可信？
这个修改是否和现有 workflow 协同？
这个修改是否让结果更容易理解和验证？
这个修改是否避免了重复输出和参数分裂？
这个修改是否诚实说明了物理近似？
```

如果答案是否定的，就不要为了增加功能而增加功能。

## 十六、参数组合回归测试规则

修改 workflow、参数构建、输出路径、正演、扫描、波场、dataset、diffraction 或 inversion 时，不能只跑默认参数。至少运行 quick parameter matrix：

```bash
python -m pytest -q
python -m pytest -q -W default
python tools/check_workflow_matrix.py --quick
```

如果修改影响异常体 shape、层状速度、扫描边界、短记录、高噪声、wavefield view 或其它极端组合，应额外运行：

```bash
python -m pytest -q -m slow
python tools/check_workflow_matrix.py --extended
```

参数矩阵的目标是捕获默认参数下隐藏不了的问题，例如 RuntimeWarning、NaN/inf、空数组、参数不同步、输出目录分裂和错误参数静默回退。业务诊断可以进入控制台摘要或 `research_report.md`，但代码 warning 不应被全局屏蔽。
