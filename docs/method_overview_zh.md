# 城市道路空洞 DAS 三维正演与定位方法说明

本文档说明本项目当前阶段的方法路线、物理含义、适用边界和结果解释原则。它面向代码阅读、组会汇报和后续真实数据接入准备。

## 1. 问题背景

城市道路下方可能存在空洞、脱空、松散回填、管沟、破碎带等浅层异常体。这些异常体会改变浅层面波传播过程，常见表现包括：

- 直达瑞雷面波能量衰减；
- 异常体附近出现绕射、散射或反射响应；
- 波形相干性降低；
- 局部频谱能量和传播速度异常。

本道路场景有一个很重要的工程特点：接收阵列并不一定是专门铺设的地震检波器，而可能是道路一侧人行道或绿化带下方已有的运营商通信光纤。通过 DAS 设备，光纤可以成为密集接收阵列。主动源则由锤击点沿道路另一侧逐点激发。

这种观测方式有成本低、布设快、接收点密集等优势，但也带来限制：光纤只在道路一侧，锤击点在另一侧，观测孔径不完整，横向位置和埋深会有耦合。因此本项目采用“三步走”路线：

1. 先做三维道路几何下的可控合成数据；
2. 再在合成数据上验证绕射/散射识别与定位算法；
3. 最后为真实道路 DAS 数据预留接口和质量控制流程。

这条路线的价值在于：先把几何、走时、可视化和定位逻辑讲清楚，再处理真实数据中的耦合、噪声、触发误差和光纤路径误差。

## 2. 三维场景定义

本项目使用如下坐标系：

- `x`：沿道路方向，也是沿光纤方向；
- `y`：横穿道路方向；
- `z`：深度方向，向下为正。

典型几何为：

- DAS 光纤位于 `y=0`；
- 锤击线位于 `y=W`，其中 `W` 是道路宽度；
- 第 `j` 个震源点为 `S_j = (x_j, W, 0)`；
- 第 `i` 个 DAS 通道为 `G_i = (x_i, 0, 0)`；
- 空洞顶部或主要散射中心为 `D = (x0, y0, h)`。

这个场景不能简单当作二维共线 MASW 测线处理。原因是：

- 震源线和接收线在 `y` 方向分离，不在同一条直线上；
- 空洞可能位于道路任意横向位置，而不是刚好在测线下方；
- 源-空洞-接收的绕射路径是三维路径；
- 单侧 DAS 孔径会使 `y0` 和 `h` 的分辨率低于 `x0`。

因此，代码中的直达波距离和绕射波距离都保留三维几何，而不是只使用 `|x_i - x_j|`。

## 3. 正演模拟方法

当前实现不是完整三维弹性波全波形正演，而是“三维等效瑞雷面波运动学/属性正演”。它关注的是：在道路几何已知、等效瑞雷波速度给定时，直达波和空洞散射波大致应该在什么时间到达 DAS 通道，以及它们的能量如何随距离、衰减和通道质量变化。

### 3.1 直达瑞雷波

直达波走时为：

```text
t_direct = t0 + |S_j - G_i| / VR
```

其中：

- `t0` 是触发时间或震源起跳时间；
- `VR` 是当前频带下的等效瑞雷波速度；
- `|S_j - G_i|` 是三维源检距。

这一步的物理意义是：锤击激发浅层面波，面波沿道路浅部传播，并被道路另一侧光纤记录。

### 3.2 空洞绕射/散射波

空洞或异常体可以被简化为一个散射中心，也可以被离散成少量等效散射点集合。对于单个散射点 `D=(x0,y0,h)`，绕射/散射走时为：

```text
t_diff = t0 + |S_j - D| / VR + |D - G_i| / VR
```

它表示能量先从震源传播到异常体，再由异常体散射到接收通道。对于 `box`、`cylinder`、`ellipsoid`、`line/zone` 等形状，当前代码只是把异常体边界或长条区域近似成若干等效散射点，然后叠加多条 `S-D-G` 路径。实际空洞散射当然比这个模型复杂：散射强度会受空洞尺度、形状、埋深、频率、材料性质和波型转换影响。当前模型故意保持简单，是为了先验证几何和定位逻辑，不代表真实弹性边界散射。

### 3.3 层状有效速度

早期版本中的层状速度模型主要用于画图展示，因此三层模型对正演记录影响不明显。当前版本增加了 `layered-effective` 模式：根据主频估计等效波长和敏感深度，

```text
lambda = VR / f
z_sensitive ≈ alpha * lambda
w(z) = exp(-z / z_sensitive)
```

再对层速度做加权调和平均得到 `VR_eff`。正演和扫描都使用同一套 `VR_eff` 计算走时。这个做法能让浅层/深层速度变化影响直达波和绕射波到时，但它仍是工程化近似，不是完整 Rayleigh 频散曲线计算，也不是三维弹性波全波形正演。

### 3.4 振幅、噪声与通道质量

正演中还简化加入了以下因素：

- 几何扩散：路径越长，振幅越弱；
- 等效衰减：用指数衰减近似频率和介质损耗；
- 尾波：用几个延迟子波模拟浅层复杂散射；
- 交通类噪声：用低频窄带干扰模拟城市环境背景；
- 弱耦合通道：随机降低部分 DAS 通道增益；
- 坏道：随机设置少量极弱通道。

这些处理不是严格物理模拟，而是为了让合成数据更接近真实 DAS 主动源数据的复杂程度。

### 3.5 为什么适合作为第一阶段原型

优点：

- 计算量小，可以快速生成大量可控数据；
- 几何关系清晰，适合教学展示和算法验证；
- 便于测试不同道路宽度、空洞位置、埋深和速度条件；
- 能直接验证定位扫描是否按三维路径工作。

局限：

- 不包含完整弹性波传播、波型转换和边界反射；
- 不描述真实瑞雷波频散；
- 不包含复杂空洞形状的完整散射场；
- 振幅只具有定性解释意义。

## 4. 定位方法原理

### 4.1 直达波速度拟合

对于每个锤击点和 DAS 通道，直达波到时满足：

```text
t_i = t0 + sqrt((x_i - x_s)^2 + W^2) / VR
```

代码首先在预测到时附近拾取直达波包络峰值，然后用三维距离拟合 `t0` 和 `VR`。这个 `VR` 是当前频带和当前浅层结构下的等效瑞雷波速度，不应解释为完整速度剖面。

### 4.2 直达波压制

空洞散射波常常贴近直达波。如果使用过宽的 mute，可能把真正的散射响应一起压掉。因此项目提供两种方式：

- 平滑 mute：简单直接，适合散射明显滞后的情况；
- 模板减去：逐道拟合直达波子波振幅并相减，更适合浅部异常导致的近直达散射。

真实数据中还需要根据触发时间、震源子波和通道耦合情况进一步改进。

### 4.3 三维绕射扫描

对每一个候选参数 `(x0, y0, h, VR)`，构造理论绕射走时：

```text
t_diff(S_j, G_i) = t0 + |S_j - D| / VR + |D - G_i| / VR
```

然后在处理后的 residual gather 中沿这条走时曲面采样能量和相干性，得到评分 `score(x0,y0,h,VR)`。评分最高的候选被认为是最可能的疑似异常中心。

当前扫描支持三种模式：

- `joint`：多炮联合评分，是默认模式；
- `single-shot`：只用指定一炮，便于观察受限孔径下单炮结果的离散性；
- `compare`：同时保存每炮最佳候选和联合结果，用于教学展示。

多炮联合有助于提高沿道路方向 `x0` 的稳定性，因为真实异常体应在多个炮点记录中产生一致的绕射走时曲线。但单侧 DAS + 对侧锤击仍是受限孔径，`y0` 和 `h` 的耦合不会因为多炮联合而完全消失。

### 4.4 为什么 y-h 容易耦合

单侧 DAS + 对侧锤击只有有限横向孔径。对某些候选点来说，改变横向位置 `y0` 和改变埋深 `h` 会产生相似的总路径长度，因此走时曲线可能很接近。这会导致：

- `x0` 通常更稳定，因为 DAS 通道沿 `x` 方向密集；
- `y0` 和 `h` 容易形成一片高分区；
- 结果更适合给“范围”，而不是给唯一精确值。

因此，扫描结果中的不确定性范围非常重要。

## 5. 结果解释原则

当前阶段输出应表述为：

- 疑似异常体位置范围；
- 置信度或相对评分；
- 不确定性范围；
- 建议复核区域。

不应直接表述为“确诊空洞”。在真实道路数据应用前，必须完成：

- 光纤路径和通道位置标定；
- 锤击点坐标与触发时间校正；
- 浅层等效速度估计；
- DAS 通道耦合质量控制；
- gauge length 和通道间距影响评估；
- 管线、井盖、路缘、车辆和施工噪声干扰核查；
- 必要时与钻探、探地雷达、地质雷达车或其他手段联合验证。

## 6. 如何阅读项目输出图

- 三维场景图：看光纤、炮线、道路宽度和异常体是否在正确几何位置；
- 平面布设图：看 `x-y` 横向孔径和空洞平面位置；
- 剖面图：看空洞埋深与道路浅层结构；
- 速度模型图：看当前使用的是均匀还是分层等效速度模型；
- shot gather：看直达波同相轴和空洞绕射/散射是否可见；
- 残差 gather：看直达波削弱后异常事件是否更突出；
- 扫描评分图：看高分区是否围绕真值或疑似异常范围；
- 动画：理解直达波前、空洞散射波前和 DAS 接收线之间的几何关系。

## 7. 后续扩展方向

要进一步逼近真实波场，可以沿以下方向扩展：

1. 2.5D/3D 弹性波正演，验证当前运动学模型的适用范围；
2. 多频带/频散建模，把 `VR` 从单一常数扩展为频率相关参数；
3. FK、扇形滤波、预测滤波等绕射增强方法；
4. 多炮联合反演和贝叶斯不确定性估计；
5. 真实 DAS 数据接口、触发时间校正和光纤路径标定；
6. 与管线、井盖、钻孔或 GPR 资料联合解释。
